# Public Redaction Rules

This repository is public. Use a privacy-first default.

## Do Not Add

- phone numbers
- street addresses
- identity documents
- private email threads
- API keys, tokens, credentials, or `.env` files
- private customer data
- private employee or stakeholder names
- internal employer documents
- confidential revenue, pricing, contract, tender, customer, or vendor details
- screenshots that reveal private data
- non-public client or employer strategy

## Safe Defaults

- use city/state only
- link to public LinkedIn/GitHub profiles
- describe work at a professional level of abstraction
- use approximate or already-confirmed metrics only
- anonymize sensitive client and project details
- keep architecture notes conceptual unless the implementation is already public

## Before Publishing A New File

Ask:

- Would this help someone impersonate me?
- Would this reveal private employer or client information?
- Would this expose a system, credential, private dataset, or security detail?
- Would I be comfortable with a recruiter, client, former colleague, or unknown visitor reading it?

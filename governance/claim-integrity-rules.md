# Claim Integrity Rules

This repo should strengthen career storytelling without drifting into unsupported claims.

## Rules

- Keep claims truthful and grounded.
- Do not invent metrics.
- Mark approximate numbers as approximate.
- Separate confirmed facts from future portfolio drafts.
- Avoid confidential proof.
- Do not name private clients, internal stakeholders, or sensitive partners without permission.
- Prefer "supported," "helped shape," "led," "built," "created," "coordinated," and "directed" only where the scope is accurate.

## Evidence Standard

Each strong public claim should have at least one of:

- resume source
- project source
- public repo evidence
- public portfolio artifact
- documented timeline
- user-confirmed metric

## Useful Claim Categories

- leadership scope
- team size
- project scope
- systems built
- platforms supported
- workflows designed
- training delivered
- tools used
- product-development cadence
